# AI 口播（数字人）方案调研

> 目标：输入一段文案 → 用自己或 AI 生成的形象 → 自动产出口播视频，**本地部署**。
> 硬件：192G 内存 + RTX 5090(32G) + RTX 4080(16G)，跑本地方案绰绰有余。
> 整理日期：2026-07-01

---

## 一、整体判断

- **上手不难**：出一个"能看、能听懂"的口播视频，几小时就能跑通。
- **难点不在能不能做**，而在做到嘴型自然、声音像真人、长视频不穿帮 —— 需要好素材 + 少量调参。
- 本地方案免月费，5090 负责推理，192G 内存跑大模型/批量渲染很舒服。

---

## 二、完整流水线（3 步）

```
你的文案 ──TTS/声音克隆──> 音频 ──数字人/对口型──> 口播视频
```

### 1. 声音（文字转语音 / 克隆自己的声音）
本地开源、中文效果好：

| 工具 | 特点 |
|------|------|
| **GPT-SoVITS** | 中文口播最常用，几十秒样本即可克隆声音，情感自然 |
| **CosyVoice 2 / Fish Speech / IndexTTS** | 音质高，中文强 |
| **F5-TTS** | 克隆快、部署简单 |

> 录一段本人 1~3 分钟的声音，即可让 AI 用"你的嗓音"念任意文案。

### 2. 形象（真人 / AI 生成）
- **A. 用自己的形象（推荐，最真实）**：拍 30 秒~1 分钟正对镜头、轻微动作的视频当底板。
- **B. AI 生成形象**：用 Flux / SDXL 生成正脸照片，或用即梦 / 可灵生成人物视频。

### 3. 对口型 / 驱动（把音频"贴"到脸上，核心步骤）

| 工具 | 出品方 | 开源 | 说明 |
|------|--------|------|------|
| **MuseTalk** | 腾讯 TMElyralab | ✅ | 快、实时性好、显存友好；`TMElyralab/MuseTalk` |
| **LatentSync** | 字节跳动 | ✅ | 音画同步更自然、清晰度更高，但更吃显存/更慢；`bytedance/LatentSync` |
| **Sonic / EchoMimic / Hallo** | — | ✅ | 一张照片 + 音频 → 会说话的视频，适合只有照片时 |
| **Wav2Lip / SadTalker** | — | ✅ | 老牌、简单，质量一般，适合练手 |

> 追质量用 LatentSync，追速度/批量用 MuseTalk，5090 两者都能跑。
> 注意：开源权重"可下载" ≠ "可随意商用"，商用前看各仓库 LICENSE 及依赖模型协议。

## 三、"长视频不穿帮"需要什么样的底板素材

底板 = 提供的原始人物视频（口型被替换，头/身体/背景/动作保留）。要点：

**画面/拍摄**
- 正对镜头，脸部占比适中，避免大侧脸、频繁扭头
- 清晰、光线均匀，别逆光/忽明忽暗（模型只重绘嘴部，光照突变会露馅）
- 分辨率 1080p 以上，脸部像素越多嘴型越好
- 背景干净固定（纯色/简单背景最稳，绿幕更好）

**动作/表情**
- 嘴巴要有自然开合、处于说话状态（全程闭嘴/抿嘴会做得别扭）
- 头部小幅自然晃动即可，别大幅点头/大笑/夸张表情
- 手势少而稳，别有手挡脸、摸嘴、喝水等遮挡

**时长/循环（关键）**
- 底板可短（30 秒~1 分钟），成片长度由音频决定
- 要能首尾平滑循环：结尾姿态 ≈ 开头姿态，长音频反复用底板才不跳
- 或直接录一段与目标时长相近的底板，避免循环
- 全程同一坐姿/景别，中途别站起、别切镜头

**声音**
- 底板原视频最好本人自然说话，有真实说话节奏，替换口型更贴

> 一句话：一段**正脸、清晰、匀光、背景干净、嘴在自然说话、头微动、能平滑循环的 1 分钟视频**，就是最省心的底板。省事就按最终稿时长真人录一条。

## 四、可直接拿来用的开源项目（一站式）

### 🥇 最对口（一站式口播数字人，本地部署）

**1. HeyGem.ai（硅基智能开源）—— 最推荐**
- 专门做口播数字人：形象视频 + 文案/音频 → 直接出对口型视频
- 声画同步好、口型自然，支持中/英/日/韩等 8 语言
- 有 **一键整合包**，Windows 免装 Docker、免配环境
- 完全本地、可批量，5090 跑很轻松
- GitHub：`GuijiAI/HeyGem.ai`

**2. AIGCPanel —— 最省心的桌面 App**
- 一站式：换口型 + 语音合成/克隆/识别 + 25+ 音视频工具
- 桌面客户端，一键导入本地模型，图形界面
- GitHub：`modstart-lib/aigcpanel`

### 🥈 更"全家桶 / 可对话"

- **Linly-Talker** —— LLM+ASR+TTS+声音克隆+对口型整合，适合能对话的数字人。`Kedreamix/Linly-Talker`
- **Pixelle-Video** —— 各环节可换模型（文案 通义/GPT/DeepSeek/Ollama + 生图 ComfyUI/FLUX + 语音 Index-TTS/Edge-TTS+克隆），适合全自动做短视频。
- **Fay** —— 数字人框架，偏虚拟主播/直播带货/互动。`xszyou/Fay`

## 四·补充 A、HeyGem（现名 Duix-Avatar）深入

> 仓库已改名 **Duix-Avatar**，仍是硅基智能同一项目：`GuijiAI/HeyGem.ai`。

### 照片能出形象吗？能，但视频更好
- 官方：**1 张照片 或 1 秒视频**即可生成形象，约 30 秒克隆形象+声音，60 秒合成 4K。
- **照片**：要求光线均匀；表情/头部动作靠模型脑补，长视频更易僵、易穿帮。
- **视频（推荐）**：建议录 8~10 秒以上，光足、背景简洁、有眨眼和轻微转头，特征全、更自然。
- 结论：照片"出模特当素材"可用于快速试玩；**正经长口播仍是真人拍短视频 >> 单张照片**。

### 底层用的开源模型
| 环节 | 模型 |
|------|------|
| 语音识别 ASR | `fun-asr`（阿里 FunASR） |
| 文字转语音 / 声音克隆 TTS | `fish-speech`（fish-speech-ziming 分支） |
| 对口型 lip-sync | 只写"计算机视觉技术"，**未点名开源模型**（应为自研/未公开权重） |

部署：Docker 容器 + NVIDIA GPU，前端 Vue，核心 C，Node.js 18。

### 能替换模型吗？基本不能（非设计目标）
- 官方**没有换模型的功能或文档**；模型打包进 Docker 镜像、与流程绑定。
- 理论上可进容器魔改换权重，但属折腾级，不是点几下能换。
- **想自由替换每个环节** → 用 **ComfyUI 工作流** 或 **Pixelle-Video**（各环节可换模型）。HeyGem 走"开箱即用、少折腾"路线，代价是不灵活。

## 四·补充 B、对口型质量对比（社区实测）

> 关键结论：**在口播场景，HeyGem 不是"图省事牺牲质量"，其嘴型自然度在开源里属第一梯队。** 质量与省事在这里不冲突。

| 维度 | HeyGem | LatentSync | MuseTalk |
|------|--------|-----------|----------|
| 嘴型同步自然度 | 🥇 最优 | 🥈 强（超过 MuseTalk） | 🥉 偏粗糙 |
| 清晰度/画质 | 高（可 4K） | 🥇 最优，几乎看不到"贴脸框" | 面部细节保存一般 |
| 头部/表情自然 | 好 | 好 | 头部自然但嘴唇糙 |
| 速度/实时性 | 一般 | 慢、吃显存 | 🥇 最快（30+fps 可实时） |
| 易用性 | 整合包开箱即用 | 需自搭 | 需自搭 |
| 主要限制 | **只能正面人脸**、机位固定、难换模型 | — | 纯口播质量最弱 |

**共识：**
- 嘴型自然度：HeyGem > LatentSync 1.5 > MuseTalk
- 画面清晰度/不露框：LatentSync 最强
- 速度/多语言实时：MuseTalk 最强

**取舍不在嘴型，在别处：**
- HeyGem 短板是"死板"（正面固定、姿态少、难换模型），不是嘴型。
- LatentSync 价值在"画质/不露框"，适合大特写、商单，但要自搭、更慢。
- MuseTalk 价值在"实时/直播/批量快"，纯出片质量不划算。

**要质量的落地顺序：**
1. 先用 **HeyGem** 出片打样，嘴型已是成片水准，快速验证内容与形象。
2. 对画质极致挑剔（大特写/商单）→ 同素材再上 **LatentSync** 对比，挑更清晰、不露框的。
3. **MuseTalk** 留给实时/直播/大批量赶量场景。
> 一句话：**HeyGem 先跑（质量第一梯队），死磕画质的镜头再上 LatentSync 补刀。**

## 五、建议的落地路线

1. **先跑通思路**：用剪映"数字人" / 腾讯智影 / HeyGen 在线平台上传文案+形象出片，当天感受效果和上限。
2. **本地一站式出片**：直接上 **HeyGem**，下整合包，当天出片（最贴合"文案+形象→视频"）。
3. **想要图形界面管理多模型** → AIGCPanel；**想要实时对话** → Linly-Talker。
4. **进阶自定义工作流**：ComfyUI 串 `GPT-SoVITS`(声音) + `LatentSync/MuseTalk`(对口型)，一次输入自动出片。

### 现实预期
- ✅ 容易：出"能看能听"的口播片，几小时跑通。
- ⚠️ 难：嘴型完全自然 + 手势表情 + 长视频不穿帮，需调参 + 好底板。**真人拍底板视频 >> 单张照片驱动。**
